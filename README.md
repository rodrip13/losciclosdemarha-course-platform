## User Activity Tracking Setup

This project includes a feature to track user activity (e.g., lesson completion, video progress) and log it to a [Supabase](https://supabase.com/) project.

To enable this feature, follow these steps:

### 1. Set up Supabase

1.  Create a new project on Supabase.
2.  Inside your project, go to the **SQL Editor** and run the following script to create the `user_events` table:

    ```sql
    -- Create the table for user events
    CREATE TABLE public.user_events (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
      user_id TEXT NOT NULL, -- Should match the type of your user IDs
      event_type TEXT NOT NULL,
      event_data JSONB
    );

    -- Optional: Add comments for clarity
    COMMENT ON TABLE public.user_events IS 'Stores user interaction events for analytics.';
    COMMENT ON COLUMN public.user_events.user_id IS 'The ID of the user performing the action.';
    COMMENT ON COLUMN public.user_events.event_type IS 'The type of event (e.g., LESSON_COMPLETE, VIDEO_PROGRESS).';
    COMMENT ON COLUMN public.user_events.event_data IS 'JSON object with specific data about the event.';
    ```

### 2. Configure Row Level Security (RLS)

This step is **critical for security**. It ensures that users can only submit their own events and cannot read any data from the table.

1.  In your Supabase project, go to **Authentication -> Policies**.
2.  Select the `user_events` table.
3.  Enable Row Level Security if it's not already enabled.
4.  Create a new policy using the following SQL script. You can do this from the UI or the SQL Editor:

    ```sql
    -- Enable Row Level Security for the table
    ALTER TABLE public.user_events ENABLE ROW LEVEL SECURITY;

    -- Create a policy that allows any user to INSERT events.
    -- Note: We are not checking user_id against auth.uid() here because
    -- the frontend is currently using a mock user ID. In a real auth system,
    -- you would add a check like: (user_id = auth.uid()::text)
    -- For now, this policy allows writing but denies all reading.
    CREATE POLICY "Allow users to insert their own events"
    ON public.user_events
    FOR INSERT
    WITH CHECK (true);
    ```
    **Important:** This policy does **not** grant `SELECT`, `UPDATE`, or `DELETE` permissions, which is intentional. Your `anon` key should only be able to insert data.

### 3. Set up Environment Variables

1.  In the root of this project, create a new file named `.env`.
2.  Add your Supabase URL and `anon` key to this file:

    ```
    VITE_SUPABASE_URL=YOUR_SUPABASE_PROJECT_URL
    VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
    ```
    You can find these keys in your Supabase project settings under **Project Settings -> API**.

Once these steps are completed, the application will automatically start sending tracking events to your Supabase database.
